创业公司应该招怎样的人

发表于 2015-01-11   |   分类于 Thinking   |  

或许用『你希望怎样的人加入你的团队』来描述这篇文章的核心会更为准确一些。

创业公司在初期遇到的最大的一个问题就是：如何招到合适的人？很多公司招人都招不到，更别说再挑三拣四的找一个合适的人了，君不见现在一个一年经验的iOS开发工程师薪资都敢要15K/月，很多公司都打破头的去抢，想要招一个双方都看着『顺眼』的，难上加难。

但是我们并不能因此就放宽招聘的条件，创业公司初期的团队一定要找到最合适的人，因为赶项目进度或者其他原因，凑合的招进来一个人，不仅发挥不了他的价值，甚至会给团队造成负面影响，所以一定要找到合适的人。

怎样才是合适的人呢？

  1. 愿意和你的团队在一起的人，不仅仅是工作上会有交流，因为人最宝贵的资产也是时间，而把时间花在与一群自己并不喜欢的人相处，是一件很浪费的事情。团队里面的每一个人都应该是独特的，而同时又应该是『相同』的，比如我们公司创始人团队一共三个人，其中两个是阿森纳的球迷，我也是阿森纳球迷，我们的第二个产品经理也是阿森纳球迷；我们热爱运动，注重身体的健康；对待中医持保留态度。并不是简单的『三观』保持一致，而是真正的朋友；
  2. 团队互补的人，绝对不要找重复的人，每个人都负责独立的一部分工作，减少内部的竞争，避免公司内部的冲突。另一方面专业的人做专业的事情，我们的产品设计到硬件，刚开始的时候我们没有懂硬件的人，每个人忙的焦头烂额，做出来的东西还都不满意，后来我们找到了一个有七八年经验的硬件产品经理，只花了三个月的时间，我们的硬件就走上了正规；
  3. 认同公司文化的人，公司的文化就是公司本身，最重要的就是你们要做的事情和你的团队，如果你们正在做的事情非常有吸引力，并且你能够清楚的表达出来，并告诉他你为什么能够成功，就能吸引到你想要的人，另一个就是你的团队，如果他们是同一类人，那就是理想的团队伙伴，因为某个大牛或者神交已久的朋友而选择某个公司的人不在少数；

其实招人要考虑的因素还是非常多的，上面三点是我觉得非常重要并且很基础的三点，其他诸如是否对创业有热情，是否无畏，是否有很强的自我学习能力，是否能够自控都要考虑进去。

一个创业公司最重要的资源就是时间，招人却又是最花费时间，而且短期内不可能完成的工作，看上去很矛盾，其实也很好解决：一是平常就做好人才的储备，这也是一个长期的过程；二是调动其他资源来推进项目的进度，之后我再详细写如何实施的问题。

如何做好创业公司的产品经理

发表于 2015-01-09   |   分类于 Thinking   |  

创业快两年了，做出来一套Android VOD系统，自身能力也获得了极大的成长，这是一段非常宝贵的经历，我也会抽出一些时间，总结这么长时间以来的一些感悟。

刚进公司的时候，是以产品经理的身份，后来VOD系统立项之后，我的title变成了项目经理，我认为创业公司项目经理和产品经理是不用分开的，最终的目的都是把产品做好，内在逻辑也是一样的，只是工作方式上有所区别。

我想先谈谈我认为创业公司好的产品经理应该具备的能力：

  1. 执行力，没有执行力，一切都是空谈。产品经理是产品的第一负责人，无论产品的任何事情都需要产品经理去推动，这需要极高的执行力；
  2. 产品的Vision，一个好的产品经理，一定要对产品有宏观的把握，为什么要做这个产品，产品现在发展到哪一个阶段了，下一步应该怎么办，会有什么变化，都要了如指掌；
  3. 学习能力，创业的过程中肯定会遇到大量的新知识，这就要求产品经理一定要有很强的学习能力，即使现在不会，也可以在很短的时间内学会并掌握。这既是挑战也是机遇，我在不到一年的时间内，不敢说对KTV行业做到了了如指掌，但也超过了这个行业95%的人，而且还学会了大量的硬件开发知识；
  4. 认识自己，人只有清醒的认识自己，知道自己现阶段的核心能力是什么，知道什么是自己能做的，什么是不能做的，才能最大限度的发挥自己的价值。专业的事情要交给专业的人，而不是遇事都逞强；
  5. 沟通，不断的和团队成员进行沟通，不仅仅是产品方向上的’还包括公司方面的，一定要做到信息透明，如果一个创业公司发生了信息沟通障碍，那还怎么发挥团队的最大战斗力。产品经理的责任就是保证产品能够做好，而沟通就是确保各个环节正常进行的润滑油；

暂时就想到了这些，之后慢慢更新，包括对产品的认识、项目的管理、自身的挑战等等。

纯粹的目标

发表于 2014-08-21   |   分类于 Thinking   |  

创业最怕的是什么？每个人都想太多。

简单的说，就是目标不纯粹，每个人做事情都会瞻前顾后。

举个简单的例子，比如说公司今年的目标是销售额一千万，分到你的部门的指标是一百万，这就是你的目标，所有的事情都应该围绕着这个目标来进行，每一件事情做不做的评判标准都应该是：对我们的目标有没有影响。
如果这个时候出现一个人或者一个事情，对目标的达成会造成负面影响，那我们就应该马上制止，并把团队拉回正规。如果你的目标不纯粹，那么你处理这些事情的时候就会考虑很多其他的因素，比如私人关系、个人利益等等，相对来说，你达成目标就会非常困难。

另外很多人执行力不够，总是找理由说想法太多不知道怎么做，其实就是没有想法不知道怎么做。

自勉。

关于低调

发表于 2014-08-17   |   分类于 Thinking   |  

今天对我影响最大的一句话就是：“过度低调沉默，是一种隐藏极深的愚蠢和无能，长远来看会让你付出巨大代价，错失许多机缘。为了逃避无足轻重的人的吹毛求疵与扭曲嫉妒，而隐藏自己的闪光点，那些真正对你工作生活有巨大影响力的人，也许就擦肩而过了。”

Trump的一本书，有一章很短但极具启发性，说的是“出席(attending)”的概念，无论你在最潦倒还是最得意的时候，都要最大化你“出席”的频率，因为出席才能带来很多巧合与机会，这背后有统计上的严谨规律。玩微博和网上社交同样有此原理，很多人在互联网上而不是现实，结识了最重要的各种工作生活关系。

这与我之前所信奉的截然相反，我之前一直觉得酒香不怕巷子深，只要自己有能力，就不怕别人发现不了你，而且低调会赢的更多的尊重，所以我基本上不写微博，不发朋友圈，博客也没有到处宣传只有几个人知道（最多写简历的时候会放上去），我期望着被人发现，从来没想到过主动“出席”，而现实是残酷的，微博只有僵尸粉才会关注我，朋友都以为我没有微信，博客从来没有回应，写的东西犹如扔进一潭死水（当然写博客的目的更多是自我总结和思考，也没指望很多人回来看，现在信息爆炸，看八卦都没时间呢）。

虽然现在心理上对于“出席”还是略有排斥，但思想上是非常认同的。人是社会性的动物，最高层次的需求就是实现自我价值，获得社会的认同。人与人也会因为相似的价值观人生观而走到一起。尤其是我现在在做互联网，应该更多的与其他人接触，相互了解，相互学习，主动出击，而不是等着别人过来找你（除非你是百年一遇的不出世的天才）。过度的低调或者沉默其实也是对自己实力的不自信。

是不是要给自己定一个目标，比如每周见一个陌生人（能够给自己带来思考的），或者之前的同事等等，这个要好好想想。

90后创业

发表于 2014-08-15   |   分类于 Thinking   |  

最近两天互联网最热门的事情应该就是IDG的“90后基金”发布会了，而我们的CEO恰巧是其中一位，我也从我所了解的方面来讲一讲为什么现在大家愿意投资90后。

投资其实就是投人，人只要靠谱，项目、方向、产品什么的都好说。而VC看重的也是90后身上具有的一些特质，不同于80后，70后的共性，比如：

  1. 基础经济条件不错；
  2. 很早的就接触到了互联网；
  3. 很多人受过国外的教育，有最好的眼界和见识；
  4. 拥有真正的世界观和价值观；
  5. 渴望自我实现；
  6. 习惯挑战权威；
  7. 年轻是最大的资本（我们CEO经常说的一句话就是：我都没时间了，其实他才92年而已）；

上面这几点在我们CEO身上表现的非常明显，其他的90后CEO也都有着相似的共性，尤其是第五点，是促使他们不断进步、不断学习的最原始动力（我们CEO早已实现财务自由，还在拼命的创业），他们是真的为了兴趣或者理想主义而坚持。

那VC投资他们，到底是投资什么呢？

  1. 极强的学习能力，能够迅速的学会自己不懂的知识和技能，知道如何获取有用的信息；
  2. 对创业是真正的兴趣，因此才愿意去学习，有足够的动力支撑自己创业，创业是最孤独的事情；
  3. 强大的执行力，遇到每一件事情能够快速的落实执行到位，创业过程中时间是最宝贵的。

创业一年了，想想自己成长最快的也恰恰是这几方面，很幸运。

对了，接下来每天最少要看100篇新闻，初期先提高阅读的质量，然后再要求速度。

比特素养

发表于 2014-05-01   |   分类于 Book   |  

一天的时间读完了『比特素养』这本书，然后发现对我自己来说没什么用，我没有获取到任何有价值的信息，这并不是说这本书写的太烂，而是这本书中所介绍的方法论我都知道，并且已经在实践了。

这本书主要介绍了一个人如何在互联网时代处理大量的信息，从邮件、待办事项、阅读信息等各个方面教你一些方法，可以看做是一个方法论的合集。

在我看来每个人都应该有自己的方法论，有自己做事情的逻辑，然后不断完善这套方法论，提高自己的效率。对于没有自己方法论的同学，可以参考一下这本书，帮助自己建立方法论。

我的方法论很简单，核心思想与GTD的理念基本保持一致，分为四个阶段：收集、计划、执行、回顾。具体运用到实际中，比如阅读，是这样的：

  1. 收集：通过RSS、微博、知乎、Twitter等信息源，获取信息，简单筛选之后，将有用的信息保存在Pocket或者Evernote中。
  2. 计划：每天花固定的时间进行深入阅读。
  3. 执行：深入阅读之后，将自己的想法整理出来，一般会写Blog或者笔记。
  4. 回顾：每个一段时间之后（一般是一个月），重新阅读之前的Blog或者笔记，看自己的想法是否和之前的一样，如果不一样，哪里发生了改变，为何改变，不断更新自己的知识库。

这套方法用到其他的领域一样可行，我用这套方法管理团队，处理个人工作计划，进行财务规划。敏捷开发其实也是这样的流程，只不过处理具体事务的时候，会根据不同的目标，不同的情景，执行的形式或者细节会有很多不同。

如何提高工作效率

发表于 2013-07-31   |   分类于 Thinking   |  

本文是对cooshell上[加班与效率](http://coolshell.cn/articles/10217.html)一文的总结,摘取了我认为对我有价值的部分.

效率是指单位时间工作创造的价值.

  1. 增加有用功.
    • 为什么增加这个需求?
    • 是否可以简化需求?
    • 业务上和用户上最大的痛点是什么?
    • 如果在10个需求内只能做3个,如何筛选?
  2. 降低总功.
    • 多做产出性的工作.
    • 减少执行成本.
    • 减少劳动密集型的工作.
  3. 形成合力.
    • 效率不是每个团队各自的效率,而是整个团队对整个产品负责的共同使命.

如何判断一个需求的优先级:

  1. 产品经理对每一条需求评估业务上的营销力.比如:XXXL影响一千万以上用户或是可以占到上亿美金的市场.而XXL则是百万用户或者千万美金的市场.然后还有XL,L,M,S等.
  2. 开发团队同样对每一条需求进行评估,要投入的人员,时间等各种成本.XXXL表示一年.XXL半年,XL3个月,L两个月,M一个月,S两周等.

所以:

  1. 业务影响力是XL,时间人员成本是S,这是最高优先级.
  2. 业务影响力是M,时间陈本是M,这是低优先级.
  3. 业务影响力是S,成本是XL,砍掉这个需求,是亏的.
  4. 业务影响力是XXL,时间成本是XXL,需要简化需求,把需求简化到XL,时间人员成本变成M以下.

学会表达

发表于 2013-07-24   |   分类于 Thinking   |  

最近发现自己很多问题,主要是表达方面的:

  1. 知识太散乱,不成体系.
  2. 词不达意,不能抓住要害.

具体表现就是,经常在和别人的辩论中,无法有理有据的说服别人,说出来的话自己都不知道什么意思,更不用说让别人理解了.虽然之前看过[金字塔原理](http://book.douban.com/subject/4882120/),但是没有经过长期有意识的训练,是无法突破的.

所以我在想我要训练自己,首先就是输入材料,有输入才有输出.手边也有非常多的好的材料可以帮助自己,比如Wiz,Pocket,Reeder,知乎等.(有的人天生过目不忘,可以旁征博引,这是天份,虽然我上初中的时候也会达到这种水平,最近5,6年退化太多了.)

输入

信息源

通过微博,Google Reader(现在改用feedly+Reeder了),知乎,Quora,Twitter等.大多数时间是快速浏览一遍,找到自己感兴趣的内容.一般在上面所列的网站中,遇到感兴趣的话题我都会重点关注,之后再根据他们的内容做筛选,过滤掉我不喜欢的内容,所以我的信息源一直是在保持动态更新的.

信息组织

经过第一步的筛选,我会将感兴趣的内容保存在Pocket或者Wiz笔记中,每天会花一个小时左右的样子,将这些文章进行整理,去掉比较水的内容,保留关键的部分.这样整个资料库的内容就不会太多.

深入的阅读

将自己整理出来的重点知识,延展开来,再次进行系统的收集资料.比如在一篇文章中提到了广告推广的CPS,那么就可以从其他的信息源中找到所有关于CPS的内容,做到系统化,而不是各种碎片.

输出

当你积累到一定的资料,你就需要输入,来锻炼你的表达能力.并不断改进你的学习方法.这样能力才会不断的进步.

知乎

知乎是一个很好的进行输出的场所,你可以获得非常多的资料,提出自己的观点,同时还可以听到其他人对你论述的评价.长时间坚持的话,会使表达能力飞速成长.同样Quora也是非常不错的,从某些方面来说比知乎还要好,前提是你的英文水平足够你在Quora上畅所欲言.

博客

博客是一个非常好的场所,它可以让你不停的思考,并反思.有的时候,你重新浏览你所有的的文章,会发现几个月之前的观点可能是错的,或者没有表述完整,这说明你进步了.博客的意义就在于将你心里所想的东西用文字表达出来,促进你去思考,不断训练你的理性思维和表达能力.

尝试与人交流

你可以尝试对你所了解事物,与其他人展开交流,不一定要争论出谁对谁错,而是看其他人是怎样思考的,他们的出发点是什么.这样的交流可以促使看问题的角度更加全面.
