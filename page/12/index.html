互联网学习的自主性

发表于 2013-06-07   |   分类于 Thinking   |  

之前看了一些关于MOOC的文章,有很多人都提出了这样的一个观点:交互和深入对话是教育的本质(也就是我之前提到过的传递价值观),而一个用户在使用MOOC学习的时候,学习过程缺乏引导,没有交互,没有同学交流,没有教师反馈,情感缺失,内容枯燥,工具用法不习惯等等均可阻碍学习者在无人监控或陪伴的情况下独立学习.这就引申出来一个问题:一个用户学习的自主性,是使用MOOC学习的前提条件,还是在学习过程中慢慢培养,作为网络学习的一个结果?

现在大部分的MOOC都是在做知识的搬运工作,将原本分散于网络的知识汇集到一个地方,为用户提供一个更方便的入口.使用MOOC进行学习的用户,大部分都有很强的自学能力,他们都能够很好的运用各种工具找到自己想要学习的知识,MOOC网站则提高了他们的学习效率.

提高工作效率的工具

发表于 2013-06-03   |   分类于 Thinking   |  

这篇博客想介绍一下我经常使用的一些工具,这些工具帮助我极大的提高了我的工作和生活效率,而且学习成本并不高.

我的电脑操作系统是Win7,大部分应用都有对应的Mac版本,或者在Mac平台有相似的应用.

Dropbox:云存储,多台电脑同步所有的文件.
Launchy:快速启动应用,可以非常快的打开应用程序和文件夹,文件等.Mac上有功能更加强大的Alfred.
VPN:你懂的.
APN:手机上使用的,尤其是iOS系统下非常的方便.
Trello:多人协作的工作平台,非常简单易用,扩展性也非常强,多人协作的利器.另外还有国外的Basecamp,和国内的Tower.这个根据每个团队的不同需求和使用方式,选择最适合自己团队的就好.
Wunderlist:个人的GTD工具,在我看来目前是Windows系统下最好的了,虽然还有很多的不足.Mac下的OmniFocus应该是这个星球上最好的GTD工具了吧.

再谈移动碎片化

发表于 2013-05-14   |   分类于 Thinking   |  

移动碎片化的本质需求应该是:帮助用户在短时间内处理大量信息.

用GTD的理念来说的话,就是手机内所有的App就是inbox,打开一个App就会收到很多未处理的信息,我们需要帮助用户快速的决定哪些信息是需要立即处理的,哪些是需要过滤掉的,哪些是要稍后处理的.

所以我们可以采用这样的一些方法:

  • 增加一个收藏按钮.
  • 可以记住上一次阅读的位置.
  • 能够审核掉垃圾内容推荐优秀.
  • 使用其他应用的分享,比如Pocket等Read it later类型的服务,或者共享到其他的社交应用,也算是变相的收藏.

这些设计可以说都是针对移动碎片化的,也可以满足我们对在手机上阅读大量内容的需求.

移动碎片化

发表于 2013-05-12   |   分类于 Thinking   |  

在很多场合听到过”移动碎片化”这个概念,经常也伴随着一些产品经理提出的解决方法,但目前看来有些方法已经不适用了.

用”碎片化”来描述移动互联网是基于以下原因:

  • 移动设备随身携带,在碎片化的时间,比如等人,上厕所等情况下,时间比较短,而且每天可能有多次这种碎片时间.
  • 一个任务没有完成的时候就被打断.
  • 手机的电量,屏幕等硬件特性决定了不可能长时间的使用.

产品的细节

发表于 2013-05-06   |   分类于 Thinking   |  

好的产品是不断打磨出来的,就像一个工匠去雕琢一件艺术品一样.

最近几个月的时间我开始学习做饭,经常在厨房中碰到各种让我使用起来很不舒服的设计,比如:

  • 电饭煲外层和内胆的颜色是一模一样的,没有辨识度.某天早上我起床后晕晕乎乎的就将大米和水全部倒进了电饭煲,却没有注意到内胆被我放在一边.此时电饭煲的内部已经被大米和水堵了一个严严实实.我又花了一天的时间拆开电饭煲进行清理,还好没有多出几个零件.
  • 电饼铛的电源开关放在机器的后面,并且和电热盘离的很近.有一次我在使用完之后去关闭电源,食指碰到电源开关的时候,拇指也被电热盘烫伤了.之后每次使用电饼铛都是小心翼翼的.其实电源开关完全可以放在电饼铛的前面.
  • 马桶和洗碗池有死角,很难清理.马桶的底座要小于马桶的上围,所以底座和墙壁之间有一个死角,是最脏也最难清理的地方.洗碗池嵌在厨房台面的拐角处,同样有清理的死角.
  • 锅盖是玻璃的,外围用钢圈包了起来,接缝出同样很难清理.
  • 地铁上的宣传片中英文不对应,很容易造成误解.中文的句子为:”突发情况请拨打110或119”,大概是这么个意思.而英文的句子为:”Please call the number 110 119”.这里少了一个单词”or”,很多人都会理解为拨打号码110119.

阅读的工具

发表于 2013-05-01   |   分类于 Thinking   |  

最近一年的时间买了很多的电子书,也越来越习惯用Kindle进行阅读.我的阅读习惯是:

  1. 第一遍通读,找到全书的框架和重点,初步判断这本书值不值得深入的阅读.
  2. 深入的阅读,多数情况下会使用脑图帮助自己整理整本书的结构.
  3. 做读书笔记,写下自己的思考.

我发现Kindle或者其他的电子设备更适合在第一遍阅读的时候使用,如果想要深入的阅读或者针对某一章节进行重点的注释,归纳和总结的话,这些电子设备使用起来并不是很顺手,经常有让人抓狂的现象.变现最突出的问题是:

  • 输入不方便,尤其是Kindle系列中没有键盘的那几款.
  • 注释没有一个集中管理的地方.
  • 页面间的跳转还是没有实体书方便,可能还是使用习惯的问题.
  • 现在大部分的电子书不管是目录还是排版都比较差
