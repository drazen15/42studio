未来的教育

发表于 2013-04-05   |   分类于 Thinking   |  

未来的教育应该是怎样的?

我们看见了未来的穿戴式计算机设备Google Glass,我们看到了未来的汽车Tesla,我们也见到越来越多的手势控制,或许不久<钢铁侠>中触摸式的计算机就会出现.

那么未来的教育应该是怎样一种形式呢?人们为什么现在有重新开始关注互联网教育取代现有的教育体制了?可能有几个原因:

  • 现在的教育面临信任危机,过高的学费,毕业生能力不足等,都让人对现在的教育产生怀疑.
  • 互联网的发展,尤其是移动互联网的发展,让人们产生了很多关于教育的想法和点子,并也已经被验证做的不错.比如:Coursera,edX,Udemy等.
  • 投资者在尝试向教育方面进行投资.
  • 环境发生变化,人们开始认可互联网教育

我们想要看清未来的教育是怎样的,首先要明白教育的本质是什么.教育的本质是提升人信息处理的能力,包括信息的获取,加工和运用.按照传授的内容不同,教育又可以分为三个级别:

  • 最高层级传授的是一种价值观,是人与人之间的互动
  • 中层是传授技能
  • 底层是传授知识

中层和底层的教育,都是传递信息,所以可以用数字化的方式来实现.而高层级的教育则必须通过人与人的交流来实现,俗话说”身正为师,学高为范”,这也应该是我们高等教育的终极目标:传递价值观.很遗憾的是我们目前的高等教育并没有做到这一点,这也是我们与国外知名大学间的本质差距.

教育本身是有五大要素的:受众,内容,工具,目标,情境,无论从那方面进行创新,都是在这五大要素的范围之内.而且这五大要素是相辅相成的.其中任何一个条件没有成熟,都是无法推动整个教育的前进的,这也许可以解释为什么过去互联网发展迅速的这十几年,教育的改革却收效甚微.比如我们之前使用过电气化的教室,这仅仅是工具的变革,教育的内容和情境却没有发生变化.当更先进的工具诞生时,我们教育的内容也会随着发生改变,进而影响教育的情境和目标,而情境和目标也会反过来影响到教育的内容,从而推动教育工具的变革.

了解了教育的本质和要素,我们可以总结出教育的目标:

  • 降低获取信息的阻力,互联网在这里居功至伟,正是因为各种各样的互联网工具,我们可以随时随地的获取各种信息
  • 提高处理信息的效率,这也是未来互联网教育的最主要的特性:为用户提供个性化的教育,有的人处理文字信息很强,有的人偏向比较抽象的图形化,有的人喜欢数学,有的人善于表达.我们的教育产品应该非常智能的满足各种人的需求,这是建立在标准的算法和海量的数据基础之上的.粉笔网的猿题库在这一方面就做的非常不错.
  • 降低信息运用的阻力,Google Glass可以帮助我们非常好的实现这一点,无论我们看到什么,Glass都可以反馈给我们已知的有效信息.另外一种就是经验的沉淀和复用,比如Quora和知乎.再进阶的可能是Google的知识图谱
  • 提升学习的动力,这方面可以通过游戏化和社交话来提高.

那么最后,线上教育是否可以完全的取代线下的教育?答案是否定的.原则上来说,线上能够解决的问题都可以放到线上来解决,线上解决不了的问题必然放到线下来解决.所以:

  • 传递价值观的教育是不会消亡的,只会越来越趋于精英教育.试问如果让你放弃你的大学四年教育,通过网络教育来完成相应的知识传递,你是否愿意?大学的氛围,同住四年的友情,同学间相互的讨论和灵感的迸发,是网络无法替代的.
  • 幼儿园是不会消亡的,幼儿园很好的解决了都市父母对孩子的托管问题
  • 各种夏令营,课外兴趣班,主要的目的是拓宽视野,增进友情.
  • 长江商学院这种精英教育

我们了解了教育的方方面面,知道了教育的关键环节,在每一个方面我们能做的事情都有很多,未来的教育方式是在每一个环节上都有质的提升.教育是一个系统化的工程,不可能通过一个产品或者一个网站就可以完成所有的事情,我们首先想好我们要做的是提高教育的哪一个方面,完成哪一个目标,然后做到极致.

信息流的分级

发表于 2013-04-02   |   分类于 Thinking   |  

这两天查询资料的时候突然发现,除了Google的搜索引擎,我用的最多的搜索工具竟然是知乎,Quora,Google Group以及豆瓣.而之前我还是Google Reader的死粉,遇到不懂的问题,就会去Reader里面搜寻答案,优质的RSS源加上Google强大的搜索,Reader成了一个巨大的知识宝库.

但随着Google宣布即将关闭Reader,以及Twitter,微博,自媒体的兴起,RSS聚合是不是穷途末路了?Google Reader的替代者会是谁呢?知乎,Quora,还是Wolfram Alpha,或者Google下一代的知识图谱?

首先让我们来搞清楚Google Reader是什么吧.

具体的RSS含义我就不解释了,维基百科上有,在这里我将信息分为3个级别,分类的依据是信息内容的深度和时效性.

  • 第一级信息,信息内容含量少,及时性高.门户网站,Twitter,微博,即时消息等属于这一类的信息.
  • 第二级信息,信息内容质量开始提升,有一定的延时性.比如:个人博客,专业论坛,知乎,Quora,Google Group.
  • 第三级信息,信息内容质量非常高,有一定的学术性.比如专业的学术论文,基础学科的研究资料等等.这类信息通常比较难以在大众中流通.

以上只是一个很粗略的分类,每一级中还有很大的不同:门户网站和微博是不一样的概念,门户网站是作为一个组织机构进行信息的发布,而微博更多的是个人.另外博客和知乎也不一样,知乎是带有筛选机制的互动问答社区.

回到正题,Google Reader应该分在哪一级别的信息流当中呢?Google Reader并不是一个简单的内容聚合器,它更多的属性应该是作为一个工具,将不同级别的信息汇总到一起,通过分享,star,follow等筛选出优质内容的一个平台.它可以做到内容的及时性,同时保证内容的深度.而Google 力推的Google+是否可以做到这一点,并取代Google Reader呢?

之前Google在它的搜索结果中已经添加了”+1”按钮,结合Google强大的搜索功能,希望籍此引入社会化的分享行为,来帮助Google筛选出更加符合用户需求的内容,虽然效果不知道怎么样,但不乏为一种思路.每一个优质的Google+用户相当于一个优质的RSS源,结合”+1”的内容,以及Google+的消息分流机制,Google+的想象空间确实要比Reader更大一些,更何况还有一个目前只是雏形的Google知识图谱.

Google+是有能力取代Google Reader的,但在Google+还没有完全成型,或者说Google+的用户数量还不是足够多的时候,就急于关掉Google Reader,是不是太莽撞了一点?起码得有个缓冲的时间吧.或许Google有自己的考虑吧.

不必猜测Google行动背后的原因,我们要做的是不断改进自己获取内容的方式,然后高效的获取优质内容.

可以这么说,Google通过它自身强大的实力做到了RSS领域当之无愧的老大,然后自己杀死了自己.虽然死的方法太愚蠢了点.

在做信息分级的时候想到了一点,就是现在被炒得火热的自媒体.作为自媒体的载体,无论是沟通的实效性,或者内容的传播上,微信都是第一选择.自媒体具有第一级信息传播快的特点,同时又具有第二级信息中的内容深度,所以个人感觉自媒体的未来应该是定位于小众,在某一领域做到绝对的权威或者有自己的独特性.否则就会淹没于浩瀚的信息洪流中.

城邦暴力团

发表于 2013-03-24   |   分类于 Book   |  

花了不到一周的时间，读完了<城邦暴力团>.

说实话,当我读到300多页的时候,已经忍不住想要摔书了,感觉完全不像是一本武侠小说,而是一个带着厚厚眼镜,穿着过膝盖大长褂的老学究写出来的八股文,这哪里是武侠啊,没有会绝世武功的高人,没有恩怨缠绵的女侠,甚至连能让人记住的武功绝学都没有.

我决定耐着性子,一目十行的翻完这本书.但随着阅读的深入,我渐渐的发现了一个不一样的江湖.

总的来说,这本书有几个特点:

  • “掉书袋”,作者旁征博引,随时可能打断之前的思路,插入一个看上不毫不相干的情节.
  • 太注重细节,一幅画,一个事物,都可以描写的非常仔细.
  • 看上去没有明显的叙事逻辑,经常在不同的角度和时间线上跳来跳去,很容易迷惑,张大春称此为松散的叙事结构 ,这倒是让我想起了陈萌荣的评书<兴唐传>.
  • 颠覆感,会告诉你另一个版本的历史.

物有两面性,有的人会将此视为新武侠的特征,有的人却受不了这样的写法,从豆瓣上的评论即可看出.如果读者不是杂学旁收,涉猎甚广,可能很难理解张大春的写作意图,从这方面来看,这本书甚至有点像张大春写给自己看得,或者是用这本书从读者中筛选出与自已一样的同类人.这本小说就像是一个引子,它包罗万象,你可以顺着书中的线索,去了解中国的传统江湖,去了解中国百年史,了解张大春的个人史.前提是你有足够的耐心和求知欲.这确实是一本非常精彩的武侠小说.

金庸曾说过:侠之大者,为国为民.整个<城邦暴力团>就是一群小人物推动的中国历史,看似不经意的一件小事,一个小人物,就像蝴蝶效应一样,改变历史的车轮.有人的地方,就有江湖,不可能有人置身事外.

所以我决定,接下来首先去通读张大春其余的作品,由书及人,然后返回来再度第二遍的<城邦暴力团>,二次阅读会做一个详细的笔记,整理思路,将第一遍没有注意到的细节,没有看明白的地方梳理清楚,甚至可以此书中提到的其他书籍或故事,找来仔细阅读.好的书每读一遍都会有新的发现,是百读不厌的.

另外,读书做笔记真的是一个好习惯,要慢慢培养.

如果你读腻了金庸,古龙,不如读读张大春的<城邦暴力团>.

辩论七律

发表于 2013-03-20   |   分类于 Thinking   |  

转自科学松鼠会

互联网拉近了人们的距离，但并没有同化人们的思想，网络辩论硝烟弥漫，热点话题层出不穷。但嘈杂背后的真相，却是情绪释放的帖子远多于冷静讲理的帖子，即使试图讲理的帖子，也有很多是逻辑混乱的，以致于各说各话，难获共识。特别是在微博，四处起火，一地鸡毛，而乱相丛生最主要的原因是网友的逻辑能力欠缺。根据多年的辩论经验我总结了七条辩论逻辑，供您参考。

###1. 有“证伪性”才有意义

你是一个战士，连长的战前动员称“只要冲锋跑得快，就不会受伤”，你拼尽全力冲了出去，被人血肉模糊地抬了回来，你质疑连长，连长的回答是“那是因为你冲得不够快，否则就不会受伤。”你喷过一口血后慢慢明白了，连长的那句动员根本没有潜在的可以被证明不对的可能性，简单地说就是没有“证伪性”，解释权完全由他，这是句永远正确的骗人鬼话。若动员改为“只要冲锋速度达到每秒5米，就不会受伤”，这就有“证伪性”了，只要有人冲锋速度超过每秒5米还受了伤，就可以证明连长的动员是错的。

在《learn python the hard way》的第三章中遇到了浮点数的概念。Google了一下，作为浮点数的入门

什么是浮点数

相对于定点数(常用来表示整数和纯小数,如:8,0.6等等),浮点数能够表示带小数部分的数字(如:12.34,939001.32,0.00023,8.0等).
定点数的缺点在于其形式过于僵硬,固定的小数点位置决定了固定位数的整数部分和小数部分,不利于同时表达特别大或者特别小的数字.而浮点数很好的解决了这一问题,计算机将浮点数分成两部分存储,一部分表示值,另一部分用于对值进行放大或者缩小.因此浮点数可以表示小数值,非常大和非常小的数值.

移动应用推送

发表于 2013-03-18   |   分类于 Work   |  

我们有一个产品叫做“每日一句”，主要的功能是:每天提供一个英语句子,很简短的,不超过7个单词,然后打乱顺序让用户重新排列,类似于非常简单的拼图游戏.除了英语之外还有日语,法语,韩语等.
我们先不说这个应用是否有存在的必要(改天说这个问题),今天老大让我去做iOS上”每日一句”的推送,每天推送当天的句子给用户,希望提高用户的活跃人数.

我本身是非常反感的:

  • 首先作为一个用户,我自己是非常反感收到一些不是那么重要的推送信息的,这会对我造成信息阻塞
  • “每日一句”作为一个工具类的应用,应该做到在特定的环境下让用户想到则合格App,举个列子:每次出去吃饭的时候都会想到大众点评.而通过推送的方式,短时间内会提高用户的活跃率,但是如果一个用户累计一周没有打开应用,看到7条推送消息,进而删掉应用也不是没有可能.
  • 目前我们的推送系统还有很大的问题,”每日一句”是一个多语种的应用,所以我们要实现精准的推送是非常难得,目前只能做到一个大概的推送,按照之前推送的效果,推送8000左右的消息,最多只有15个打开率.在没有获得精准的用户信息的情况下,盲目推送只会适得其反.
  • 老大用其他应用的列子来证明,推送会提高活跃率.推送可以提高活跃率我不否认,但是对于不同的应用,得到的效果是不同的.之前已经证明过”每日一句”的效果并不好,为什么还要继续推送.

其实推送系统是一个已经用烂的东西,在使用这个推送之前最好考虑一下我要用推送系统达到什么目的,这个目的是否和我应用的核心目的一样.

我和老大对于推送系统不同看法的根本原因就是:我们对于一个产品的理念不同,老大很看重下载量和活跃率等数据,只要应用可以用,就立马采取一些市场推广的手段来增加应用的下载量和活跃率.

而我则是希望可以改进应用的质量,在此基础之上再做推广,而一些功能是否添加取决于这个功能是否符合我们应用的核心诉求.比如说”每日一句”这个应用是希望利用碎片时间帮助用户提高英语水平,通过推送能否直接或者间接达成这个目标?我的答案是否定的.所以我拒绝推送,但是我又不得不执行.唉.

Launchy Pro

发表于 2013-03-18   |   分类于 Life   |  

iOS系统比较蛋疼的就是文件夹，对于我这样一个有整理洁癖的人来说，iOS上的App管理非常的不方便，一个文件夹可以放的App数量有限，如果我有很多同一类的App，一个文件夹放不下是让我很头疼的事情，另一方面，很多个文件夹搜索起来也是很费力的。而且一直不习惯手机上的键盘输入，打字效率很低，所以springboard的作用也无从发挥。

后来发现了Launchy Pro这个应用，一见倾心，马上购买。使用之后发现果然是神奇。现在我将Launchy Pro放在doc上面，然后其他的App随意分文件夹，只要通过Launchy管理就好，效率非常高。

Launchy的可扩展性非常高，根据个人的使用习惯可以有不同的组合。目前大部分应用都已经支持url scheme，可以添加进入Launchy，少部分国内的应用还是没有支持。不过我们可以通过手动添加的方式将一些应用加入Launchy。

如何找到App的url scheme?

  • 首先可以Google一下,关键词为 App name+url+scheme
  • 在Handleopenurl上查找
  • 通过其他的方法,比如说网易新闻客户端,在找了很长时间无果之后,我返回网易新闻客户端的首页,找到了以下几个关键词:
    • 手机下载链接: http://3g.163.com/\newsapp,关键词:newsapp
    • 下载的安卓apk文件,关键词:netease_newsreader
      由此我可以推出4个url scheme,一个一个试下来,最后正确的是:newsapp://

以上是非官方获取url scheme的方法.

我亲测可用的url scheme：

  • 网易新闻客户端: newsapp://
  • 淘宝客户端: taobao://
  • wunderlist客户端: wunderlist://
  • 知乎客户端:zhihu://

Trello官方回复说近期内会加上url scheme.
