Leader的作用

发表于 2012-09-25   |   分类于 Work   |  

上周末的时候参加了公司组织的拓展活动，这是我第二次参加类似的活动，同时也是我第二次以队长的身份带领队伍获得拓展训练的第一名。这两次拓展训练带给我最深的思考就是：一个队伍中的leader，他的作用是什么？

一个队伍中经常会有各种各样的人，实干者、思考者、创作者等等若干角色，而一个leader并不一定是在某一方面表现的特别突出的，但一定是可以将所有成员凝聚在一起，充分发挥每一个队员实力的人。

所以在我看来，一个队伍的了leader本质上是一个为所有队员服务的人。leader充分了解每一个队员的特长与不足，他知道某一个队员在某一个位置可以发挥出超常的实力，他也知道某一个队员可能会出现问题，需要队长的及时帮助。leader具有极强的分析问题能力和总结能力，当困难出现时，可以为队员指明前进的方向，在所有队员都发表自己见解的时候，能够很快的拿出一个令人信服的解决方案。leader最后一个享受胜利的果实，但遇见困难时确实第一个冲在最前方。leader并不是对所有领域的知识都精通，但所有队员都信任他，尊重他的每一个决定，而且这种信任和尊重是相互的。

在项目开发中，产品经理扮演的就是这样的一个leader角色。产品经理本身不对任何人发号施令，而是通过沟通说服其他人接受自己的意见。

产品经理是最了解产品的人，同时也是不断推动项目前进的人。产品经理更多的是服务于整个开发团队，从而服务于产品。如何更好地为团队做好服务，是做好产品的关键。

搬个梯子

发表于 2012-09-18   |   分类于 Life   |  

之前给自己制定了学习计划，还起了一个非常霸气的名字，叫做”诺曼底登陆“，到今天已经是第十二周了，但是完成的进度……惨不忍睹啊。所以今天在这里重新将整个计划的重点贴出来，用来监督自己完成。

计划的核心主要是用来提高自己的学习能力，养成良好的学习习惯。

  • 每天晚上11：30之前睡觉，早上5：30之前起床
  • 每天坚持跑步至少半小时，其他锻炼半小时
  • 每天阅读一小时，阅读各种书籍
  • 每天学习一小时的英语，主要途径为沪江网校的课程
  • 每天学习一小时的python编程。可以先从VBA入手
  • 坚持撰写博客

另外一部分是关于自己情商部分的：

  • 不盘算太多，凡事尽力而为，顺其自然
  • 不刻意的对其他人好，也不刻意的对其他人坏，多思考，多静心
  • 控制脾气，不情绪化
  • 相信自己比依赖他人更重要，要强大自己的内心
  • 保持自己的人格力量
  • 包容，换位思考
  • 停止抱怨。世界上的事情本来就是这样的，你要做的是适应，而不是抱怨
  • 挑战自我舒适区，希望用3个月来养成习惯

其实这是我很早就给自己定下来的计划，但对于我这样一个有拖延症的人来说……上面所提到的事情，有的我正在做，有的还没有开始，从今往后，我会用博客记录下我前进道路上的一点一滴，记录下我的成长。我会将我遇到的困难说出来，也会将我成长的经验分享出来。

一年前大学刚毕业的时候，对所有的事情都没有感觉，接触的互联网也没有现在这么深，也没有思考过自己的人生。工作一年了，自己的世界观和价值观发生了很大的改变，从那时起开始萌生了出国的想法，即使去不了美利坚、德国，能去香港和新加坡也是不错的。

出国最主要的原因并不是为了自己，而是为了自己的家人，为了爱我的人，为了我的孩子，即使我在国内工作，我也希望我的家人生活在国外，百姓网面试我的时候曾经问过我，我理想中的生活是怎样的？我当时回答说：我的家庭健康幸福，这就是我最大的成就。

小脑残们

发表于 2012-09-12   |   分类于 Life   |  

看到这样一句话：

小脑残们都有一个毛病，先把一个人想象成什么样的人，然后攻击谩骂这个他想象中的人

很是赞同。

我觉得造成这种事情的主要原因有两个：没有自己独立的思考和没有充分的沟通交流（或许小脑残们根本不愿交流，他们完全充分相信自己的判断，即使错了也不愿意承认，永远都有狡辩的理由）。

生活中存在很多这样的人，小脑残们看到一件事情以后，不愿意去思考这件事情背后隐藏着什么，或者说是完全站在自己的角度，将这件事情套进自己的生活，然后就会觉得：哦，原来是这样的，怪不得。但是所有的这一切过程都是在小脑残们的脑袋里进行的，根本没有给其他人任何交流的机会，他完全按照他的生活轨迹，就已经将整个事情定性了。

如果小脑残们平常比较宅比较腐、或者脑残片看多了，那么他们看待所有的事情都像是在迫害自己。办公室里，你无心的说了一句好冷啊，小脑残们会觉得你是在怪他开空调；吃饭的时候你说了一句好贵啊又不好吃，小脑残们会觉得你怎么这么抠门。哈哈，很有意思的一件事情。

其实做产品也一样，如果没有对这个产品进行思考，没有接触真实的用户，那么这个产品知识你想象的产品，完全不是用户的需求。

社会化

发表于 2012-09-10   |   分类于 Thinking   |  

最近总是听到很多人在说社会化，社会化的营销、社会化的平台、社会化的产品，但到底什么是社会化？很多人并不能说清楚。真是的现状是，大家只是把社会化当成一个噱头，很多社会化的产品前景并不明朗，已经死掉的也有很多。

现在有很多产品是在做垂直领域的社会化产品，比如旅游，母婴，生活信息，学习等。那么这些领域适不适合做社会化产品？社会化的产品应该具有什么样的属性？

我觉得要做垂直领域的社会化产品，产品应该具有以下属性:

  • 产品使用频率高，使用频率的的产品适合做成工具，而不适合社会化
  • 产品有内容沉淀，内容是用来支撑整个社交平台的，所以从这方面来说我觉得汽车领域不适合做社会化，因为可以把车主联系到一起的无非是有关车的方方面面，这些内容更多的是车主自身的需要，很难产生互动，所以汽车领域更多的是可以做工具类应用。
  • 产品是不是一个刚需，只有刚需才有社会化的必要，如果不是刚需，如何让用户长时间的投入到社区中？

由此看来，我觉得图书、音乐、电影、教育等领域是可以做社会化的，豆瓣做的就非常好，格瓦拉也不错。粉笔网就是才出来的教育领域的社交平台，稍后我会做一个关于粉笔网的详细分析。

纪念在我生活中消失的人

发表于 2012-09-07   |   分类于 Life   |  

晚上的时候看了一个视频，《罗纳尔多在国家队的最后四分钟》，是巴西队的一场国际友谊赛，罗纳尔多在上半场41分钟左右的时候替换弗雷德上场，直到比赛结束，总共不到5分钟的时间，罗纳尔多在国家队的最后时间。

最喜欢的两个前锋，亨利和罗纳尔多，罗纳尔多让我结识了足球，亨利让我彻底爱上了阿森纳。98年的时候只知道有一个齐达内和罗纳尔多，02年的时候完整的看了一场世界杯，从此爱上了足球，巴拉克的坚强，克洛泽的头球帽子戏法，科科的头缠绷带，伊尔汗挑球过卡洛斯，里瓦尔多的夸张表演，卡恩的神级发挥，韩国队的无耻，都给我留下了深刻的印象。但是只有一个人是完全的征服了我，那就是罗纳尔多。

那个时候罗纳尔多真的是外星人，风驰电掣的速度，灵巧的过人，无敌的射术，你能想象他来自地球吗？

10年后再看到罗纳尔多站在球场上，已经发福的身体，跑两三步就气喘吁吁，只能站着喘气。没有了速度，还有超强的意识，短短的四分钟，队友不停的做球给罗纳尔多，罗纳尔多也获得了几次非常好的机会，但不是打在门将的身上，就是打飞。以往进球如草芥，现在却不知道如何将球送进空门。中场休息的时候，所有队员站成两排向罗纳尔多致敬，眼泪忍不住就流了下来。

这些年我们送走了多少儿时的偶像，内斯塔、马尔蒂尼、巴乔、巴蒂斯图塔、卡恩、巴拉克、维埃拉、耶罗、罗纳尔多、莱曼、皮雷、永贝里，许许多多的人，还有一些人虽然仍在球场上奔跑，但不知何时就会消失在我们的视线内。

这就是成长，不停的有人进来，不停的有人出去，被记住了，被遗忘了，你无法改变什么，你被时间的双手不停的向前推，当你想回头的时候，发现身后一片空白，所有的东西都被遗忘。

所以，当你在一所及其不靠谱的学校学习一门不靠谱的专业，当你毕业为了生存不得不远走他乡，当你第一次租房被房东讹了工资，当你遇见极品的室友的时候，你为什么要抱怨呢？一些注定不会在你人生中再次出现的人，你又为什么因为他们愤怒？时间终究会让你坦然面对一切，也会证明你的价值，成长的道路会让你经历一切，会让你失去很多，会让你忘掉很多人，这是成长的代价，也是你的财富。

所以不要抱怨，就像感谢罗纳尔多带给你的感动那样，感谢所有你遇到的人，他们让你成长，他们构成了你的回忆，不管以后记不记得，至少他们在你的生活里出现过。这就够了

多问自己why

发表于 2012-09-05   |   分类于 Work   |  

无论做什么，都应该多思考，一个事情从何处来，又往何处去，将会发生什么样的结果，要有打破沙锅问到底的精神，不停的问自己why，只有思考才能不断的进步。

下午的时候，一直在研究timeline的形式，对比了新浪微博和twitter，感觉信息的传播真是一门高深的学问。之后又试用了最近新出的教育方面的垂直网站–粉笔网，发现新浪微博和粉笔网存在一个问题：我和某一个人的相互@和评论很容易淹没在信息流中，如果稍后我想重新回顾这段对话，如何整合之前的信息是一个比较头疼的问题，而私信由于私密和权限设置的关系，在这方面并没有发挥作用。新浪采取的方法是在评论里面显示对话，有一个缺点就是整个对话必须是连续的，一旦有一个断点，整个对话就不能完整显示。

这时我很兴奋，马上想到一个解决方案，就是将我与某人所有的@和评论单独拉出一个页面来显示，这样可以很方便的去查看所有的信息。冷静了一下，我有问了我一个问题，用户在什么场景下会使用这个功能？这个功能是否会频繁使用到？它可以带给整个产品新的体验吗？

我一一解答了以上问题：

  • 当两个用户在多条有相互关联性的信息内发生多次互动的时候，会需要这个功能，这个时候不如采用更加方便的私信
  • 频繁使用的前提是，两个用户之间的互动必须很多，而且都是公开的有价值的信息，从这方面来看，粉笔网可能更需要这个功能
  • 微博更多关注的是信息的流动，而这个功能只能说满足了部分人的小需求，对于整个产品的核心体验并没有改善，甚至让产品显得臃肿。

所以这个功能就是一个鸡肋。

做产品的时候多问问自己why，不仅仅是避免过度的对产品做加法，更重要的是帮助你思考，这才是最有意义的。

2012-09-03工作总结

发表于 2012-09-03   |   分类于 Work   |  

上周的时候工作上出现了一些疏漏，对我现在所负责的产品造成了比较大的影响。在此检讨一下。

情况是这样的，我们在上上周五的时候，也就是2012-08-24，在app store上更新了我们的产品，周末的时候发现有一个重大bug导致软件无法正常运行，而这个bug在之前的测试过程中竟然没有发现。上周一的时候我们修复了bug并且马上提交了更新，苹果的审核速度大家都知道的，我们也只能等。我向上级汇报了这个情况，只是说出现一个比较严重的bug，并且我们提交了新的版本，审核通过立马上线。

上级当时也没意识到这个bug会对我们造成什么影响。从上周一开始我就一直在关注用户的反馈，首先是itunes的评论出现大量一星的评价，占当前版本总评价的80%以上，主要问题就是那个bug，到周五的时候一星的差评之增加了少量。直到这个时候，我还是简单的认为一些差评并不会对我们的用户群体造成很大的损害，而且这些差评在我们更新版本之后会重新变为五星，所以并没有很重视，而是按照之前的计划继续进行开发。

周末的时候，上级发现了用户给予我们的大量差评，认为这是我们整个工作流程的重大失误。此时我才意识到问题的严重性，确实，会在itunes上进行评论的只是一小部分用户，大部分用户是直接删除应用。这个bug对我们造成的影响可能一时半会看不出来，但很快就会反应在数据上面，更重要的是我们失去了很多忠实用户，这才是最宝贵的。

总结一下出现这个问题的原因：

  • 作为一个产品经理，首先我负有不可推卸的责任，我应该是整个测试流程的第一关，是产品的核心用户，但是我没有发现这个比较严重的问题
  • 没有严格要求测试，对于测试反馈的结果也只是简单掠过，整个测试过程仿佛走了一个流程，没有起到实质的作用，这在产品开发过程中是不可想象的，而我却犯了一个这么低级的错误
  • 测试没有一个完整的流程，完全是测试人员自己把控，发现问题完全随机，这个已经制定了新的方案去解决
  • 发现问题后没有及时向上级反馈，更没有将问题的严重性描述清楚，而是将解决问题的方法寄望于快速更新。如果及时反馈，可能会获得更好的解决方法，比如在app描述中添加警告，阻止用户进行更新等
  • 需要更好的用户沟通平台。可以考虑微信的公共帐号

成长的代价比较惨痛，同时也惊醒自己以后不要犯类似的错误。

慢一些

发表于 2012-09-01   |   分类于 Thinking   |  

每次过马路的时候，总是有一种冲动要去闯红灯，特别是车辆较少的时候，如果有人先踏出了这一步，那我十有八九是跟着闯了红灯，当我发现自己的这个问题之后，立马做出了改正，哪怕没有一辆车的时候也坚决不闯红灯，闯红灯不仅是对自己的不负责，更是对他人生命的漠视。诚然，不闯红灯不代表不会出事，总会有七十码之类的事情出现，但这并不是自己可以犯错的接口。

后来我观察到其实不止是我，非常非常多的人都有闯红灯的习惯，随处可见在车流中不停穿梭的人们，我们真的是赶时间吗？恐怕仅仅浪费在微博上的时间就足够我们等一个小时的红灯了。那是什么原因让我们总是愿意冒着生命的危险去通过闯红灯的方式去节省那一分钟的时间？

中国的教育鼓励从小就开始竞争，小学的时候争当班干部，家长也把还在当成相互攀比的对象，上了大学所有人挤破头的进学生会，长期的环境熏陶，我们无论做什么其实内心都在和周围的人进行攀比，当你穿过红灯的时候是不是有一种异样的成就感？你在等红灯的时候看到别人穿越红灯会不会非常的焦灼？从另一个方面来看，闯红灯其实就是中国社会集体浮躁的一个缩影，每个人都在拼命的追赶其他人，任何时候都希望利用自己的小聪明超过其他人，但很多时候却是我们的肉体穿过了红灯，灵魂却车流阻断，可望而不可及。

另一方面源于集体对规则的无视，经常可以看到n多的汽车堵在一个路口，原因就是其中某一辆汽车没有遵守交通规则，其他的汽车不愿想让造成的。在中国需要遵守的只有潜规则，没有规则。当大家都遵守潜规则的时候，闯红灯自然而然的就发生了。

有的时候需要我们慢一些，多审视自己，加强内心的修炼，内心强才是真的强，希望有一天可以看到，即使在没有红灯的情况下，大家依然都能够谦让。
