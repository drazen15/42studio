为什么要用OKR

发表于 2022-03-09   |   分类于 Work   |  

上周正式在组内推行OKR，我提前申明了两点：1. 和绩效、奖金无关；2. 目的是让大家更加清晰透明的知道我们公司、产品的目标，方便大家协同，团队的小伙伴对OKR接受度还挺好的，截止到今天，大家都已经确定好各自的OKR，并进行了拉齐。

我个人一直在用GTD的方式管理任务，年初的时候也会列一个全年的个人目标，然后拆分到GTD的待办事项，某种程度也算是OKR。之前我也经历过各种各样的考核方式，比如传统的KPI，和流水挂钩的绩效（运营销售属性强的公司，产品技术考核流水就很奇怪），还有阿里著名的375，那为什么我会在新公司新的团队推行OKR呢？核心是：希望提高每一个同事的ROI。

  1. OKR更加符合我们当前公司、团队的阶段，符合我的管理风格，我们是初创公司，目前没有很强的业绩流水压力，我的管理方式也更加注重结果、关注过程，所以明确的O对我们团队来说非常重要；

  2. 可以让小伙伴专注于做更加重要的事情，围绕着O来展开各项工作，不迷于当下，不流于空想；

  3. 给予小伙伴更强的个人意义和成就感，激发他们的自主性，OKR要求目标必须是自发性、有鼓舞性、有挑战性的，我也一直在给小伙伴们强调工作中个人的成长是最重要的，要把自己的成长和公司的成长相结合，自己的目标和公司的目标拉齐，这才是对自己最有利的工作方式；

  4. 更好的协同，每个人的OKR都是透明的，也经过拉齐，理解别人的利益才能找到更好的合作方式，才能共赢；

OKR并不是万能的，就像有了GTD的工具，并不能保证可以合理安排个人待办事项一样，一方面个人要有意愿去用，知道工具对个人的价值，另一方面，要会用、善用工具，有一定的方法论做为支撑。

几个职业建议

发表于 2022-02-21   |   分类于 Work   |  

几个来自于Nicholas C. Zakas的职业建议：

  1. 不要只是做执行，别人要你怎么做，你就怎么做；
  2. 学会推销自己，通过各种渠道，各种方法；
  3. 要学会如何做一个团队leader；
  4. 生活最重要，家人最重要；
  5. 自己找到属于自己的道路；
  6. 时刻把自己当做owner，不管是公司还是产品；
  7. 和优秀的人一起共事；

另外补充几个自己总结的：

  1. 学好英语；
  2. 要有非常清晰的工作目标和方向；
  3. Just do it；
  4. 完成比完美更重要；

抄袭「Notion」的「wolai」

发表于 2022-02-14   |   分类于 Work   |  

V站上看到一个帖子，讨论wolai抄袭notion的。

顺手去看了一下wolai的产品，然后看到了官网上关于wolai融资的一篇文章，通篇90%内容讲的是自己之前创业经历，恨不得讲出来自己所有认识的互联网大佬，只在最后简单讲了下自己关于这个产品的最初想法。再去搜索了工商信息，发现是马锐拉创业做的。

单纯产品上来讲，我很鄙视这种像素级的copy，完全没有自己独立的产品愿景和使命。
写融资的文章全都在写自己，赖好讲讲自己拿到钱之后产品的规划吧。
不过中国自古信奉「成王败寇」，wolai做得好，估计大家也就不记得Notion了，君不见「Whats app」和「Kik messager」已消失在历史长河，众人只知道「小而美」的绿色小软件。
